# **1.1 INTRODUCTION** {#1.1-introduction}

The purpose of this test plan is to outline the strategy, objectives, scope, and approach for testing the Clean City web app, which allows users to schedule waste pickups based on their preferred date and time.

Our main objective is to ensure that all features work as expected, provide a smooth user experience, and meet functional and non-functional requirements.

# **1.2 Objectives** {#1.2-objectives}

* Verify that the registration and login feature works effectively.  
* Verify that the scheduling feature allows users to book, modify and cancel waste pickups successfully.  
* Ensure that notifications, confirmations, and rescheduling options function correctly.  
* Ensure that the site is responsive, has a friendly user interface, high performance, and portability.  
* Identify and log any bugs, errors, or usability issues encountered during testing.

# **1.3 Scope** {#1.3-scope}

## **1.3.1 In Scope:** {#1.3.1-in-scope:}

- [ ] User registration and login.  
- [ ] Scheduling, ability to edit and cancel pickups.  
- [ ] Notifications i.e emails or SMS.  
- [ ] Ability to track and manage pickups.  
- [ ] Site performance.  
- [ ] Profile Management  
- [ ] Feedback feature  
- [ ] Payment Processing

## **1.3.2Out of scope:** {#1.3.2out-of-scope:}

- [ ] GPS tracking for collection vehicles.  
- [ ] Integration with external payments.
